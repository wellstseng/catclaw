# Skill：<skill-name>（Generator）

> 依固定模板 + 風格指南產出 `<產出類型>`，確保每次格式一致。

## 觸發

- 使用者要求產出 `<產出類型>`（不限措辭，含「寫一份 X」「生成 X」「整理成 X」皆觸發）

## 核心鐵則

1. **照模板填空，不自由發揮** — 禁止改變章節順序、新增/省略段落
2. **缺資訊就問，不要腦補** — 用 AskUserQuestion 補齊變數
3. **風格依 references/style-guide.md，不靠記憶**

## 工作流程

### Step 1：載入風格指南
- Read `references/style-guide.md`
- 完成判定：理解語氣、格式、長度、稱呼規範

### Step 2：載入模板
- Read `assets/template-<產出類型>.md`
- 完成判定：辨識出所有 `<變數>` 佔位符

### Step 3：收集變數
- 列出模板內所有 `<變數>` 缺哪些 → AskUserQuestion 一次問完
- 完成判定：所有變數有實際值（不是「待補」）

### Step 4：填模板
- 逐節填入變數值，**禁止增刪章節**
- 完成判定：所有 `<變數>` 已替換

### Step 5：輸出
- 寫入 `<指定路徑>`
- 回報使用者：產出路徑 + 一段話摘要

## 必備檔案

| 檔 | 用途 |
|----|------|
| references/style-guide.md | 語氣 / 格式 / 長度規範 |
| assets/template-<產出類型>.md | 結構骨架，含 `<變數>` 佔位符 |

## 反模式

- ❌ inline 寫整段模板進 SKILL.md
- ❌ 變數缺漏自行腦補
- ❌ 自由增刪章節（破壞格式一致性）
- ❌ 風格漂移（每次語氣不同）

## 注意事項

- 模板修改後**所有歷史產出都會「不符模板」** — 變更模板需審慎
- 多語言產出：每語言一個 template-<lang>.md，禁止「模板 + LLM 翻譯」
